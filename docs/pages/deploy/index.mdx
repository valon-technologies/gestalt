# Deploy

Toolshed compiles to a single statically-linked Go binary and ships as a minimal Docker image. It runs anywhere you can run a container or a Linux binary.

## Resource requirements

| | Minimum | Recommended |
|---|---|---|
| CPU | 1 vCPU | 2 vCPU |
| Memory | 256 MB | 512 MB |
| Disk | 1 GB (SQLite) | Managed database |

## What every deployment needs

1. **Toolshed image or binary.** Build from `toolshed/Dockerfile` or compile with `CGO_ENABLED=0 go build ./cmd/toolshed`.
2. **Config YAML file.** Mounted or bundled. See [Config File](/reference/config-file) for the schema.
3. **Environment variables for secrets.** The config supports `${VAR}` placeholders expanded at startup. Never hardcode secrets.
4. **A database.** SQLite with a persistent volume, Postgres, MySQL, or any [supported datastore](/reference/built-in-plugins#datastores).
5. **A stable `base_url`.** Toolshed derives OAuth callback URLs from this. It must be the externally-reachable URL.

## Environment variables

### Server

| Variable | Purpose | Required |
|---|---|---|
| `TOOLSHED_BASE_URL` | External URL for OAuth callbacks | Yes |
| `TOOLSHED_ENCRYPTION_KEY` | Symmetric key for encrypting stored tokens. Generate: `openssl rand -base64 32` | Yes |
| `TOOLSHED_SESSION_SECRET` | Signs session cookies. Generate: `openssl rand -base64 32` | Yes |

### Auth

| Variable | Purpose | Required |
|---|---|---|
| `GOOGLE_CLIENT_ID` | Google OAuth client ID | If using Google auth |
| `GOOGLE_CLIENT_SECRET` | Google OAuth client secret | If using Google auth |

### Datastore

| Variable | Purpose | Required |
|---|---|---|
| `DATABASE_URL` | Connection string for Postgres, MySQL, or other SQL datastores | If not using SQLite |

### Integrations

Each integration needs its own OAuth credentials. Follow the pattern `<INTEGRATION>_CLIENT_ID` and `<INTEGRATION>_CLIENT_SECRET`.

## Minimal production config

```yaml
server:
  port: 8080
  base_url: "${TOOLSHED_BASE_URL}"
  encryption_key: "${TOOLSHED_ENCRYPTION_KEY}"

auth:
  provider: google
  config:
    client_id: "${GOOGLE_CLIENT_ID}"
    client_secret: "${GOOGLE_CLIENT_SECRET}"
    session_secret: "${TOOLSHED_SESSION_SECRET}"

datastore:
  provider: postgres
  config:
    dsn: "${DATABASE_URL}"
```

## Validate before deploying

```sh
toolshed config check -config your-config.yaml
```
