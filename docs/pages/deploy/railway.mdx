# Deploy to Railway

Run Toolshed on Railway by connecting your GitHub repository. Railway detects the Dockerfile and provisions infrastructure through its dashboard.

## Prerequisites

- A [Railway](https://railway.app) account
- Your Toolshed repository on GitHub

## Deploy from GitHub

1. Create a new project in the Railway dashboard.
2. Select **Deploy from GitHub repo** and connect your repository.
3. Set the **Root Directory** to `toolshed/` so Railway finds the Dockerfile.
4. Railway detects the Dockerfile and begins the first build automatically.

<Callout>
  Do not deploy yet — the app will crash without a database and environment variables. Cancel the initial deploy or let it fail, then continue with the steps below.
</Callout>

## Add a database

1. In your Railway project, click **New** → **Database** → **PostgreSQL**.
2. Railway creates the database and exposes a `DATABASE_URL` variable. Copy it or reference it directly from the Toolshed service.

To link the variable:

1. Open the Toolshed service settings.
2. Under **Variables**, add `DATABASE_URL` and set it to `${{Postgres.DATABASE_URL}}` (Railway's variable reference syntax).

## Set environment variables

Add these variables in the Toolshed service's **Variables** tab:

```
TOOLSHED_BASE_URL=https://<your-railway-domain>.up.railway.app
TOOLSHED_ENCRYPTION_KEY=<openssl rand -base64 32>
TOOLSHED_SESSION_SECRET=<openssl rand -base64 32>
GOOGLE_CLIENT_ID=<your-google-client-id>
GOOGLE_CLIENT_SECRET=<your-google-client-secret>
DATABASE_URL=${{Postgres.DATABASE_URL}}
```

Generate the encryption and session keys locally:

```sh
openssl rand -base64 32
openssl rand -base64 32
```

## Configure

Railway does not support mounting config files directly. Embed the config inline using an environment variable and a startup command, or commit a production config to the repo.

The simplest approach is to commit a config file at `toolshed/config.railway.yaml`:

```yaml
server:
  port: 8080
  base_url: "${TOOLSHED_BASE_URL}"
  encryption_key: "${TOOLSHED_ENCRYPTION_KEY}"

auth:
  provider: google
  config:
    client_id: "${GOOGLE_CLIENT_ID}"
    client_secret: "${GOOGLE_CLIENT_SECRET}"

datastore:
  provider: postgres
  config:
    url: "${DATABASE_URL}"
```

Then override the container start command in Railway's service settings:

```
/toolshed --config /build/toolshed/config.railway.yaml
```

<Callout>
  The path depends on how Railway copies your repo into the build context. Check the build logs for the exact working directory if `/build/toolshed/` does not resolve.
</Callout>

## Verify

Railway assigns a public URL automatically. Find it in the service's **Settings** → **Networking** → **Public Networking**.

```sh
curl -s https://<your-railway-domain>.up.railway.app/health
```

You should see `{"status":"ok"}`. Open the URL in a browser to sign in.

## Custom domain

1. In the service settings, go to **Networking** → **Custom Domain**.
2. Add your domain and follow the DNS instructions Railway provides.
3. Update `TOOLSHED_BASE_URL` to the new domain.
