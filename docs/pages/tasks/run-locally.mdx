# Run Locally

## Fastest Path

```sh
gestaltd
```

This creates `~/.gestalt/config.yaml` when needed and starts a local server on
port `8080`.

## Explicit Local Config

Save this as `openapi/httpbin.yaml`:

```yaml
openapi: 3.0.3
info:
  title: HTTPBin
  version: 1.0.0
servers:
  - url: https://httpbin.org
paths:
  /ip:
    get:
      operationId: get_ip
      responses:
        "200":
          description: OK
```

Save this as `config.yaml`:

```yaml
server:
  port: 8080
  base_url: http://localhost:8080
  encryption_key: dev-only-change-me

auth:
  provider: none

datastore:
  provider: sqlite
  config:
    path: ./gestalt.db

integrations:
  httpbin:
    plugin:
      openapi: ./openapi/httpbin.yaml
      openapi_connection: public
      default_connection: public
      connections:
        public:
          mode: none
          auth:
            type: none
```

Start it:

```sh
gestaltd init --config ./config.yaml
gestaltd serve --locked --config ./config.yaml
```

## Inspect The Server

```sh
curl http://localhost:8080/health
curl http://localhost:8080/api/v1/httpbin/get_ip
gestalt --url http://localhost:8080 integrations list
```

## Local Packaged Plugin Development

When you are iterating on a provider or runtime plugin, use `command` first:

```yaml
integrations:
  example:
    plugin:
      command: ./bin/example-provider
      config:
        greeting: hello
```

That avoids repackaging on every change.
