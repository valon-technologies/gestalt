# Use with MCP

Connect Toolshed to AI assistants like Claude Code or Claude Desktop via the Model Context Protocol.

## Prerequisites

- A running Toolshed server with at least one integration.
- A user account (sign in via the web UI or CLI first).
- An API token for that user.

## 1. Create an API token

Using the CLI:

```sh
tshed tokens create --name "mcp-token"
```

Or via the HTTP API:

```sh
curl -X POST http://localhost:8080/api/v1/tokens \
  -H "Authorization: Bearer $SESSION_TOKEN" \
  -d '{"name": "mcp-token"}'
```

Save the returned token value. You will not be able to retrieve it again.

## 2. Enable MCP in your config

Add an `mcp` block to your YAML config:

```yaml
mcp:
  enabled: true
```

To expose only a subset of integrations, add a `providers` list:

```yaml
mcp:
  enabled: true
  providers:
    - linear
    - slack
```

Omit `providers` to expose all configured integrations.

## 3. Start the server

```sh
toolshed serve -config config.yaml
```

The MCP endpoint is available at `/mcp` on the same HTTP server as the rest of the API.

## 4. Configure Claude Code

Add the following to your Claude Code MCP settings (`.mcp.json` in your project root or `~/.claude.json`):

```json
{
  "mcpServers": {
    "toolshed": {
      "url": "http://localhost:8080/mcp",
      "headers": {
        "Authorization": "Bearer your-api-token"
      }
    }
  }
}
```

Claude Code will connect to the MCP endpoint and discover the available tools.

## 5. Configure Claude Desktop

Add the same block to your Claude Desktop config (`~/Library/Application Support/Claude/claude_desktop_config.json` on macOS):

```json
{
  "mcpServers": {
    "toolshed": {
      "url": "http://localhost:8080/mcp",
      "headers": {
        "Authorization": "Bearer your-api-token"
      }
    }
  }
}
```

## Minimal example config

A complete config file with MCP enabled for a single integration:

```yaml
auth:
  provider: oidc
  config:
    issuer_url: https://accounts.google.com
    client_id: ${GOOGLE_CLIENT_ID}
    client_secret: ${GOOGLE_CLIENT_SECRET}

datastore:
  provider: sqlite
  config:
    path: ./toolshed.db

server:
  dev_mode: true
  encryption_key: ${TOOLSHED_ENCRYPTION_KEY}

integrations:
  linear:
    openapi: https://raw.githubusercontent.com/linear/linear/master/packages/sdk/src/schema.graphql
    client_id: ${LINEAR_CLIENT_ID}
    client_secret: ${LINEAR_CLIENT_SECRET}
    allowed_operations:
      search_issues: Search issues in Linear
      create_issue: Create a new Linear issue

mcp:
  enabled: true
  providers:
    - linear
```

## Verify it works

Once your MCP client connects, you should see the integration operations listed as available tools. For the config above, the tools would be:

- `linear_search_issues`
- `linear_create_issue`

If tools do not appear, check that:

1. The server is running with `mcp.enabled: true` in the config.
2. Your API token is valid and included in the `Authorization` header.
3. The user who owns the token has connected the integration (completed the OAuth flow).
4. The integration has `allowed_operations` defined.
