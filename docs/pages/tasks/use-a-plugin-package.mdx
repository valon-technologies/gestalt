# Use A Plugin Package

Gestalt can load plugin packages from a local directory, a local archive, an
HTTPS archive, or a versioned source.

## Local Directory

```yaml
integrations:
  slack:
    plugin:
      package: ./dist/slack
```

## Local Archive

```yaml
integrations:
  slack:
    plugin:
      package: ./dist/slack.tar.gz
```

## Remote Archive

```yaml
integrations:
  slack:
    plugin:
      package: https://downloads.example.com/slack-0.1.0.tar.gz
```

Only HTTPS archives are supported.

## Versioned Source

```yaml
integrations:
  slack:
    plugin:
      source: github.com/valon-technologies/gestalt/slack
      version: 0.0.1-alpha.7
```

## Prepare And Serve

```sh
gestaltd init --config ./config.yaml
gestaltd serve --locked --config ./config.yaml
```

`init` extracts or resolves the package into `.gestalt/plugins/...` and records
the prepared state in `gestalt.lock.json`.

## Package A Plugin Yourself

```sh
gestaltd plugin package --input ./my-plugin --output ./dist/my-plugin.tar.gz
```

See [Plugin Manifests](/reference/plugin-manifests) for the package layout.
