# Config File

This page documents the current `config.yaml` shape used by `gestaltd`.

## Complete Example

```yaml
server:
  port: 8080
  base_url: https://gestalt.example.com
  encryption_key: ${GESTALT_ENCRYPTION_KEY}
  api_token_ttl: 30d

auth:
  provider: oidc
  config:
    issuer_url: https://login.example.com
    client_id: ${OIDC_CLIENT_ID}
    client_secret: ${OIDC_CLIENT_SECRET}
    redirect_url: https://gestalt.example.com/api/v1/auth/login/callback

datastore:
  provider: postgres
  config:
    dsn: ${DATABASE_URL}

secrets:
  provider: env
  config:
    prefix: GESTALT_SECRET_

telemetry:
  provider: stdout

integrations:
  notion:
    display_name: Notion
    description: REST and MCP access to Notion
    mcp_tool_prefix: notion_
    plugin:
      openapi: ./openapi/notion.yaml
      mcp_url: https://mcp.notion.com/mcp
      mcp_connection: MCP
      headers:
        Notion-Version: "2026-03-11"
      connections:
        MCP:
          mode: user
          auth:
            type: mcp_oauth
      allowed_operations:
        search:
          alias: search
        pages.retrieve:
          alias: get_page

  slack:
    display_name: Slack
    plugin:
      source: github.com/valon-technologies/gestalt/slack
      version: 0.0.1-alpha.7

runtimes:
  notifier:
    providers: [slack]
    plugin:
      package: ./dist/notifier-runtime
    config:
      channel: alerts

bindings:
  inbound-hook:
    type: webhook
    providers: [slack]
    config:
      path: /incoming
      provider: slack
      operation: chat.postMessage
      auth_mode: signed
      signing_secret: ${WEBHOOK_SECRET}

  support-proxy:
    type: proxy
    providers: [slack]
    config:
      path: /proxy/slack

egress:
  default_action: deny
  policies:
    - action: allow
      subject_kind: system
      subject_id: support-proxy
      host: slack.com
      path_prefix: /api

ui:
  plugin:
    package: ./dist/custom-ui
```

## Global Rules

- `${VAR}` expansion happens before YAML decoding.
- `${VAR_FILE}` reads the file contents and trims trailing newlines.
- Unknown fields are rejected.
- Relative paths are resolved from the config file directory.
- `server.port` defaults to `8080`.
- `secrets.provider` defaults to `env`.
- `telemetry.provider` defaults to `stdout`.

## Top-Level Keys

| Key | Purpose |
| --- | --- |
| `server` | HTTP listener settings, callback base URL, encryption key, API token TTL. |
| `auth` | Platform auth for the user of Gestalt. |
| `datastore` | Storage for users, integration tokens, API tokens, and staged connections. |
| `secrets` | Runtime secret lookup for features such as egress credential grants. |
| `telemetry` | Logging and OpenTelemetry export. |
| `integrations` | Providers exposed over HTTP, the UI, and `/mcp`. |
| `runtimes` | Long-lived background plugins. |
| `bindings` | Extra HTTP routes for inbound webhooks or outbound proxying. |
| `egress` | Outbound HTTP allow or deny policy plus credential grants. |
| `ui` | Optional packaged replacement for the embedded web app. |

## `server`

```yaml
server:
  port: 8080
  base_url: https://gestalt.example.com
  encryption_key: ${GESTALT_ENCRYPTION_KEY}
  api_token_ttl: 30d
```

| Field | Purpose |
| --- | --- |
| `port` | HTTP listen port. Defaults to `8080`. |
| `base_url` | Public base URL used for callback URLs and secure cookie behavior. |
| `encryption_key` | Required when platform auth is enabled. |
| `api_token_ttl` | Optional lifetime for API tokens. Supports Go durations such as `24h` and day values such as `30d`. |

## `auth`

```yaml
auth:
  provider: oidc
  config:
    issuer_url: https://login.example.com
    client_id: ${OIDC_CLIENT_ID}
    client_secret: ${OIDC_CLIENT_SECRET}
```

Built-in auth providers:

- `none`
- `local`
- `google`
- `oidc`

The `config` block is provider-specific. See [Built-in Plugins](/reference/built-in-plugins)
for the available providers.

## `integrations`

Every integration has one `plugin` block plus optional presentation metadata.

```yaml
integrations:
  github:
    display_name: GitHub
    description: Public GitHub operations
    icon_file: ./icons/github.svg
    mcp_tool_prefix: github_
    plugin:
      auth:
        type: none
      openapi: ./openapi/github.yaml
      allowed_operations:
        repos.get:
          alias: get_repo
```

### Integration Fields

| Field | Purpose |
| --- | --- |
| `display_name` | UI and API display name override. |
| `description` | UI and API description override. |
| `icon_file` | SVG icon path resolved from the config directory. |
| `mcp_tool_prefix` | Prefix for tools exposed on `/mcp`. |
| `plugin` | The actual provider definition. Required. |

### `integrations.*.plugin`

The same `plugin` block supports inline providers, packaged providers, and
hybrid providers.

#### Package Source Fields

| Field | Purpose |
| --- | --- |
| `command` | Execute a local binary directly. |
| `package` | Load a local directory, local archive, or HTTPS archive. |
| `source` | Resolve a versioned plugin source during `init`. Format: `github.com/<org>/<repo>/<plugin>`. |
| `version` | Required with `source`. Invalid with `command` or `package`. |
| `args` | Extra process arguments for `command`. |
| `env` | Extra environment variables for external plugins. |
| `config` | Provider-specific config passed into packaged providers. |

`command`, `package`, and `source` are mutually exclusive.

#### Inline Provider Fields

| Field | Purpose |
| --- | --- |
| `auth` | Auth definition for the implicit plugin connection. |
| `params` | Connection parameters for the implicit plugin connection. |
| `connections` | Named connections for inline integrations. |
| `openapi` | OpenAPI document URL or local file path. |
| `graphql_url` | GraphQL endpoint. |
| `mcp_url` | Upstream MCP endpoint. |
| `base_url` | Base URL for declarative REST operations. |
| `headers` | Static headers applied to spec-backed or declarative requests. |
| `managed_parameters` | Fixed header or path values injected before exposure. |
| `operations` | Inline declarative REST operations. |
| `response_mapping` | Data and pagination mapping for spec-backed providers. |
| `allowed_operations` | Surface allowlist plus `alias` and `description` overrides. |
| `openapi_connection` | Named connection to use for OpenAPI. |
| `graphql_connection` | Named connection to use for GraphQL. |
| `mcp_connection` | Named connection to use for upstream MCP. |
| `default_connection` | Default connection used when the caller does not specify one. |
| `mcp` | Expose packaged provider operations on `/mcp`. |

Inline integrations omit `command`, `package`, and `source`.

### Auth Shapes Inside `plugin`

Top-level `plugin.auth` and named connection auth blocks use the same shape:

```yaml
auth:
  type: oauth2
  authorization_url: https://accounts.example.com/oauth/authorize
  token_url: https://accounts.example.com/oauth/token
  client_id: ${CLIENT_ID}
  client_secret: ${CLIENT_SECRET}
  scopes:
    - read
    - write
```

Supported auth types in config today:

- `none`
- `oauth2`
- `manual`
- `bearer`
- `mcp_oauth`

### Named Connections

```yaml
plugin:
  openapi: https://api.example.com/openapi.json
  mcp_url: https://mcp.example.com/mcp
  openapi_connection: api
  mcp_connection: mcp
  default_connection: api
  connections:
    api:
      mode: user
      auth:
        type: oauth2
        authorization_url: https://accounts.example.com/oauth/authorize
        token_url: https://accounts.example.com/oauth/token
        client_id: ${CLIENT_ID}
        client_secret: ${CLIENT_SECRET}
    mcp:
      mode: user
      auth:
        type: mcp_oauth
```

Connection modes:

- `none`
- `user`
- `identity`
- `either`

### Declarative Operations

```yaml
plugin:
  auth:
    type: none
  base_url: https://httpbin.org
  operations:
    - name: get_ip
      method: GET
      path: /ip
    - name: get_headers
      method: GET
      path: /headers
```

Each operation accepts:

- `name`
- `description`
- `method`
- `path`
- `parameters`

Parameter locations can be `query`, `body`, or `path`.

### Managed Parameters

```yaml
plugin:
  openapi: https://api.example.com/openapi.json
  managed_parameters:
    - in: header
      name: X-API-Version
      value: "2026-04-01"
    - in: path
      name: account_id
      value: primary
```

Managed parameter locations can be `header` or `path`.

### Response Mapping

```yaml
plugin:
  graphql_url: https://api.example.com/graphql
  auth:
    type: bearer
  response_mapping:
    data_path: data.items
    pagination:
      has_more_path: pageInfo.hasNextPage
      cursor_path: pageInfo.endCursor
```

## `runtimes`

Gestalt does not ship built-in runtime types. Use a packaged runtime plugin.

```yaml
runtimes:
  notifier:
    providers: [slack, notion]
    plugin:
      package: ./dist/notifier-runtime
    config:
      channel: alerts
```

| Field | Purpose |
| --- | --- |
| `providers` | Providers whose capabilities should be visible to the runtime. |
| `plugin` | Packaged runtime plugin. |
| `config` | Runtime-specific config passed to the plugin. |

## `bindings`

Built-in binding types are `webhook` and `proxy`.

### Webhook Binding

```yaml
bindings:
  inbound-hook:
    type: webhook
    providers: [slack]
    config:
      path: /incoming
      provider: slack
      operation: chat.postMessage
      auth_mode: signed
      signing_secret: ${WEBHOOK_SECRET}
```

Webhook config fields:

- `path`
- `provider`
- `instance`
- `operation`
- `auth_mode`
- `signing_secret`
- `signature_header`
- `user_header`
- `trusted_sources`

`auth_mode` can be `public`, `signed`, or `trusted_user_header`.

### Proxy Binding

```yaml
bindings:
  support-proxy:
    type: proxy
    providers: [support]
    config:
      path: /proxy/support
```

Proxy bindings currently require only `config.path`.

## `egress`

`egress` controls outbound HTTP from bindings and upstream clients.

```yaml
egress:
  default_action: deny
  policies:
    - action: allow
      subject_kind: system
      subject_id: support-proxy
      host: api.example.com
      path_prefix: /v1
  credentials:
    - secret_ref: secret://support-api-token
      auth_style: bearer
      subject_kind: system
      subject_id: support-proxy
      host: api.example.com
      path_prefix: /v1
```

### Policy Fields

- `action`
- `subject_kind`
- `subject_id`
- `provider`
- `operation`
- `method`
- `host`
- `path_prefix`

### Credential Grant Fields

- `secret_ref`
- `auth_style`
- `subject_kind`
- `subject_id`
- `operation`
- `method`
- `host`
- `path_prefix`

## `ui`

Use `ui.plugin` to replace the embedded web UI.

```yaml
ui:
  plugin:
    source: github.com/acme/plugins/gestalt-ui
    version: 1.2.0
```

`ui.plugin` supports `package`, `source`, and `version` in the same way as
other packaged plugins.
