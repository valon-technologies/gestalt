# Plugin Sandboxing

`allowedHosts` is declarative egress policy, not a security boundary by itself.
It only becomes an enforceable boundary for executable plugins when Gestalt runs
those plugins inside a sandboxed execution environment that can reliably mediate
filesystem access and outbound network access.

## Current state

Today, executable plugin sandboxing is implemented with host-local OS features
plus a local proxy:

- Darwin uses `sandbox-exec`
- Linux uses Landlock for filesystem restrictions and a localhost proxy for
  outbound HTTP mediation

That works when the underlying runtime supports those primitives. It is not yet
an architecture-agnostic sandbox story, because support varies by kernel,
container runtime, and hosting platform. A deployment can therefore accept
`allowedHosts` in config while still lacking a complete enforcement boundary for
executable plugins.

## What "complete" means

The feature is only complete when executable plugins run in an actual sandboxed
provider runtime whose isolation story does not depend on incidental support
from the host OS. In practice, that means:

- `allowedHosts` stays declarative in the config file
- the sandboxed provider runtime owns outbound mediation
- the sandboxed provider runtime owns filesystem/process isolation
- Gestalt can apply the same model across supported deployment targets without
  changing the security contract

Until that exists, host-local wrappers should be treated as best-effort
enforcement, not the portable security boundary for executable plugins.

## Operator guidance

If you need a hard boundary today, do not rely on `allowedHosts` alone for an
executable plugin. Use a deployment boundary that is independent of the main
`gestaltd` process, for example:

- a dedicated service or worker for the plugin
- platform-native sandboxing at the container or VM layer
- deployment-level egress controls such as VPC, firewall, or proxy policy

For plugin packages that need those guarantees, the preferred long-term shape is
an actual sandboxed provider runtime rather than more host-specific branching in
`gestaltd`.
