# Docker Images

Gestalt publishes two Docker images, both available for `linux/amd64` and `linux/arm64`.

## `valontechnologies/gestaltd`

The server image. Runs the HTTP API, web UI, and MCP endpoint.

| Property | Value |
| --- | --- |
| Image | `valontechnologies/gestaltd:latest` |
| Base | Alpine with `ca-certificates` and a shell |
| Port | `8080` |
| Entrypoint | `/gestaltd` |
| Default command | `serve --locked --config /etc/gestalt/config.yaml --artifacts-dir /data` |
| Config path | `/etc/gestalt/config.yaml` |
| Data directory | `/data` (writable, used for SQLite and prepared artifacts) |
| User | `nobody:nobody` |

The image is not zero-config. Mount or bake a config file before starting it. See [Docker deployment](/deploy/docker) for usage examples and production patterns.

### Tags

- `latest`
- `<version>` (e.g. `0.0.1-alpha.6`)

## `valontechnologies/gestalt`

The CLI client image. Connects to a running `gestaltd` server for authentication, invocation, and management.

| Property | Value |
| --- | --- |
| Image | `valontechnologies/gestalt:latest` |
| Base | Distroless (`gcr.io/distroless/cc-debian12`) |
| Entrypoint | `/gestalt` |
| User | `nonroot:nonroot` |

The image does not include a shell. Use `--help` to inspect available commands.

### Tags

- `latest`
- `<version>` (e.g. `0.0.1-alpha.6`)

### Environment variables

| Variable | Purpose |
| --- | --- |
| `GESTALT_URL` | Base URL of the `gestaltd` server. |
| `GESTALT_API_KEY` | API token for authentication. |

### Usage

```sh
docker run --rm \
  -e GESTALT_URL=https://gestalt.example.com \
  -e GESTALT_API_KEY=gst_api_... \
  valontechnologies/gestalt:latest \
  integrations list
```

### CI pipelines

The client image works well in CI environments where installing the native binary is inconvenient:

```yaml
jobs:
  check-integrations:
    runs-on: ubuntu-latest
    steps:
      - run: |
          docker run --rm \
            -e GESTALT_URL="${{ vars.GESTALT_URL }}" \
            -e GESTALT_API_KEY="${{ secrets.GESTALT_API_KEY }}" \
            valontechnologies/gestalt:latest \
            integrations list --format json
```
