# Authentication

This page covers building custom authentication providers. If you only need to
configure platform login for a deployment, use [Providers > Authentication](/providers/authentication).

Authentication providers handle platform login. When a user visits the Gestalt web UI or hits the HTTP API without a valid session, the authentication provider drives the login flow, validates the callback, and returns an authenticated identity. Gestalt then issues a session cookie and manages session lifetime on its own.

Beyond the core login flow, an authentication provider can optionally validate externally issued tokens (so API clients can present a bearer token directly instead of going through the browser) and control session TTL.

The examples on this page use a Google OAuth provider as the running example.

## Manifest

An authentication provider manifest declares `kind: authentication`. You can optionally include a `configSchemaPath` so Gestalt validates provider config at startup.

```yaml
kind: authentication
source: github.com/your-org/authentication/google
version: 0.0.1
displayName: Google Authentication
description: Authenticate users with Google OAuth.
spec:
  configSchemaPath: ./authentication_config.json
```

## Interface

The preferred interface is `Configure`, `BeginAuthentication`, and `CompleteAuthentication`.
`BeginAuthentication` returns the authorization URL that Gestalt redirects the user to, plus any opaque provider state that needs to survive until callback completion.
`CompleteAuthentication` receives the full callback query parameters and returns the authenticated identity.

Deprecated aliases `BeginLogin` and `CompleteLogin` still work during the migration window, and token-only validation can still be exposed through `ValidateExternalToken`, but new providers should implement `Authenticate` when they need direct bearer or request validation.

    ```go
    package googleauth

    import (
    	"context"
    	"fmt"

    	gestalt "github.com/valon-technologies/gestalt/sdk/go"
    	"golang.org/x/oauth2"
    	googleoauth "golang.org/x/oauth2/google"
    )

    type GoogleAuth struct {
    	clientID     string
    	clientSecret string
    }

    func New() *GoogleAuth { return &GoogleAuth{} }

    func (g *GoogleAuth) Configure(_ context.Context, _ string, config map[string]any) error {
    	g.clientID, _ = config["clientId"].(string)
    	g.clientSecret, _ = config["clientSecret"].(string)
    	if g.clientID == "" || g.clientSecret == "" {
    		return fmt.Errorf("clientId and clientSecret are required")
    	}
    	return nil
    }

    func (g *GoogleAuth) BeginAuthentication(_ context.Context, req *gestalt.BeginAuthenticationRequest) (*gestalt.BeginAuthenticationResponse, error) {
    	cfg := &oauth2.Config{
    		ClientID:     g.clientID,
    		ClientSecret: g.clientSecret,
    		RedirectURL:  req.CallbackUrl,
    		Scopes:       []string{"openid", "email", "profile"},
    		Endpoint:     googleoauth.Endpoint,
    	}
	return &gestalt.BeginAuthenticationResponse{
    		AuthorizationUrl: cfg.AuthCodeURL(req.HostState, oauth2.AccessTypeOffline),
    	}, nil
    }

    func (g *GoogleAuth) CompleteAuthentication(ctx context.Context, req *gestalt.CompleteAuthenticationRequest) (*gestalt.AuthenticatedUser, error) {
    	// Exchange req.Query["code"] for a token, then fetch userinfo.
    	return &gestalt.AuthenticatedUser{
    		Subject:       "user-123",
    		Email:         "user@example.com",
    		EmailVerified: true,
    		DisplayName:   "Example User",
    	}, nil
    }
    ```

    ```python
    import gestalt

    class GoogleAuth(gestalt.AuthenticationProvider):
        def configure(self, name: str, config: dict) -> None:
            self.client_id = config["clientId"]
            self.client_secret = config["clientSecret"]

        def begin_authentication(
            self,
            request: gestalt.BeginAuthenticationRequest,
        ) -> gestalt.BeginAuthenticationResponse:
            auth_url = (
                "https://accounts.google.com/o/oauth2/v2/auth"
                f"?client_id={self.client_id}"
                f"&redirect_uri={request.callback_url}"
                f"&state={request.host_state}"
                "&response_type=code"
                "&scope=openid+email+profile"
            )
            return gestalt.BeginAuthenticationResponse(authorization_url=auth_url)

        def complete_authentication(
            self,
            request: gestalt.CompleteAuthenticationRequest,
        ) -> gestalt.AuthenticatedUser:

            return gestalt.AuthenticatedUser(
                subject="user-123",

                display_name="Example User",
            )
    ```

    ```rust
    use std::sync::Mutex;
    use gestalt::{
        AuthenticationProvider,
        AuthenticatedUser,
        BeginAuthenticationRequest,
        BeginAuthenticationResponse,
        CompleteAuthenticationRequest,
    };

    #[derive(Default)]
    pub struct GoogleAuth {
        client_id: Mutex<String>,
        client_secret: Mutex<String>,
    }

    #[gestalt::async_trait]
    impl AuthenticationProvider for GoogleAuth {
        async fn configure(
            &self, _name: &str, config: serde_json::Map<String, serde_json::Value>,
        ) -> gestalt::Result<()> {
            *self.client_id.lock().unwrap() = config
                .get("clientId").and_then(|v| v.as_str()).unwrap_or("").to_string();
            *self.client_secret.lock().unwrap() = config
                .get("clientSecret").and_then(|v| v.as_str()).unwrap_or("").to_string();
            Ok(())
        }

        async fn begin_authentication(
            &self,
            req: BeginAuthenticationRequest,
        ) -> gestalt::Result<BeginAuthenticationResponse> {
            let client_id = self.client_id.lock().unwrap().clone();
            Ok(BeginAuthenticationResponse {
                authorization_url: format!(
                    "https://accounts.google.com/o/oauth2/v2/auth\
                    ?client_id={client_id}&redirect_uri={}&state={}&response_type=code\
                    &scope=openid+email+profile",
                    req.callback_url, req.host_state,
                ),
                provider_state: vec![],
            })
        }

        async fn complete_authentication(
            &self,
            req: CompleteAuthenticationRequest,
        ) -> gestalt::Result<AuthenticatedUser> {
            // Exchange req.query["code"] for a token, then fetch userinfo.
            Ok(AuthenticatedUser {
                subject: "user-123".into(),
                email: "user@example.com".into(),
                email_verified: true,
                display_name: "Example User".into(),
                ..Default::default()
            })
        }
    }

    gestalt::export_authentication_provider!(constructor = GoogleAuth::default);
    ```

    ```ts
    import { defineAuthenticationProvider } from "@valon-technologies/gestalt";

    let clientId = "";
    let clientSecret = "";

    export const provider = defineAuthenticationProvider({
      async configure(_name, config) {
        clientId = config.clientId as string;
        clientSecret = config.clientSecret as string;
      },
      async beginAuthentication(request) {
    ```

## Optional interfaces

Two optional interfaces extend the base contract.

**Authenticator** lets API clients present a bearer token or other external auth input directly instead of going through the browser flow. In Go, implement the `Authenticator` interface on your provider. In Python, inherit from `gestalt.Authenticator`. In Rust, override the `authenticate` method; the runtime still bridges the deprecated `validate_external_token` method for token-only providers. In TypeScript, provide the optional `authenticate` handler to `defineAuthenticationProvider`.

**SessionTTLProvider** controls session lifetime. The default is 24 hours. In Go, implement the `SessionTTLProvider` interface. In Python, inherit from `gestalt.SessionTTLProvider`. In Rust, override the `session_ttl` method, which returns `None` (the 24-hour default) by default. In TypeScript, provide the optional `sessionSettings` handler to `defineAuthenticationProvider`.

## Config Reference

To use a Google authentication provider in a Gestalt deployment, reference it as a named entry in `providers.authentication`:

```yaml
providers:
  authentication:
    google:
      source: ./google-authentication/manifest.yaml
      config:
        clientId: ${GOOGLE_CLIENT_ID}
        clientSecret:
          secret:
            provider: default
            name: google-client-secret
        allowedDomains:
          - example.com
```

The deployment YAML does not need to change for the `BeginLogin` to `BeginAuthentication` rename. The interface migration is inside the authored provider runtime contract, not the host config shape.

`source: ./manifest.yaml` points at the authentication provider's `manifest.yaml` during development. For production, use a published `provider-release.yaml` metadata URL. The `config` block is passed to `Configure` at startup and validated against the config schema if one was declared in the manifest.

Structured secret refs like `clientSecret.secret.provider: default` are resolved through the configured secret manager before reaching the provider. Environment variable placeholders like `${GOOGLE_CLIENT_ID}` are expanded during config loading. See [Configuration](/configuration) for details on both.
