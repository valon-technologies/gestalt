# File API

This page covers building custom File API providers. If you only need to
configure Blob/File storage for a deployment, use
[Providers > File API](/providers/fileapi).

File API providers back plugin-facing binary object storage. Gestalt adopts the
[W3C File API](https://www.w3.org/TR/FileAPI/) so plugins can work with
familiar `Blob` and `File` semantics instead of a Gestalt-specific abstraction.
That makes the same provider contract suitable for object stores, local
filesystems, document backends, and content-addressed storage layers.

## Manifest

A File API provider manifest declares `kind: fileapi`:

```yaml
kind: fileapi
source: github.com/your-org/fileapi/object-store
version: 0.0.1
displayName: Object Store File API
description: Blob/File provider backed by an object store.
spec:
  configSchemaPath: ./fileapi_config.json
```

## Interface

The File API surface maps directly onto Blob/File operations from the W3C
specification:

| RPC | W3C equivalent |
| --- | --- |
| `CreateBlob` | [`Blob()`](https://www.w3.org/TR/FileAPI/#constructorBlob) |
| `CreateFile` | [`File()`](https://www.w3.org/TR/FileAPI/#file-constructor) |
| `Stat` | reading `Blob` / `File` metadata |
| `Slice` | [`Blob.slice()`](https://www.w3.org/TR/FileAPI/#slice-method-algo) |
| `ReadBytes` | [`Blob.arrayBuffer()`](https://www.w3.org/TR/FileAPI/#blob-methods-and-parameters) |
| `OpenReadStream` | [`Blob.stream()`](https://www.w3.org/TR/FileAPI/#blob-section) |
| `CreateObjectURL` | `URL.createObjectURL()` |
| `ResolveObjectURL` | host-side lookup of a created object URL |
| `RevokeObjectURL` | `URL.revokeObjectURL()` |

Implementations are free to choose how they store bytes and metadata, but they
should preserve the observable Blob/File contract: immutable object contents,
stable object IDs, file metadata where applicable, and correct `slice` and
stream semantics.

    ```python
    import gestalt
    from gestalt.gen.v1 import fileapi_pb2 as pb

    class MemoryFileAPI(gestalt.FileAPIProvider):
        def __init__(self) -> None:
            self._objects: dict[str, bytes] = {}

        def configure(self, name: str, config: dict) -> None:
            # Validate config and initialize the backing client.
            return None

        def CreateBlob(self, request, context):
            object_id = "blob-1"
            data = b"".join(
                part.bytes_data if part.WhichOneof("kind") == "bytes_data" else b""
                for part in request.parts
            )
            self._objects[object_id] = data
            return pb.FileObjectResponse(
                object=pb.FileObject(
                    id=object_id,
                    kind=pb.FILE_OBJECT_KIND_BLOB,
                    size=len(data),
                    type=request.options.mime_type,
                )
            )

        def ReadBytes(self, request, context):
            return pb.BytesResponse(data=self._objects[request.id])

        # Implement CreateFile, Stat, Slice, OpenReadStream,
        # CreateObjectURL, ResolveObjectURL, and RevokeObjectURL too.
    ```

    ```ts
    import { defineFileAPIProvider } from "@valon-technologies/gestalt";

    export const provider = defineFileAPIProvider({
      displayName: "Object Store",
      description: "Example File API provider",
      async configure(_name, config) {
        // Validate config and initialize the backing client.
      },
      async createBlob(parts, options) {
        const id = crypto.randomUUID();
        const bytes = Buffer.concat(
          parts.map((part) => {
            if (part.kind === "string") return Buffer.from(part.value);
            if (part.kind === "bytes") return Buffer.from(part.value);
            throw new Error("blob references omitted from example");
          }),
        );
        return {
          kind: "blob",
          id,
          size: bytes.byteLength,
          type: options.mimeType ?? "",
        };
      },
      async createFile(fileBits, fileName, options) {
        return {
          kind: "file",
          id: crypto.randomUUID(),
          size: 0,
          type: options.mimeType ?? "",
          name: fileName,
          lastModified: options.lastModified ?? Date.now(),
        };
      },
      async stat(id) {
        return { kind: "blob", id, size: 0, type: "" };
      },
      async slice(request) {
        return { kind: "blob", id: request.id, size: 0, type: request.contentType ?? "" };
      },
      async readBytes(id) {
        return new Uint8Array();
      },
      async openReadStream(id) {
        async function* chunks() {
          yield new Uint8Array();
        }
        return chunks();
      },
      async createObjectURL(id) {
        return `fileapi:${id}`;
      },
      async resolveObjectURL(url) {
        return { kind: "blob", id: url.replace(/^fileapi:/, ""), size: 0, type: "" };
      },
      async revokeObjectURL(_url) {},
    });
    ```

## Plugin SDK

Plugins consume bound File API providers through the SDK transport. If a plugin
binds exactly one File API, use the default socket env var. Otherwise, connect
by binding name.

    ```go
    bytes, err := blob.Bytes(ctx)
    ```

## Config reference

Reference a File API provider under `providers.fileapis`, then bind it into the
plugins that should see it:

```yaml
providers:
  fileapis:
    assets:
      source:
        path: ./fileapi/object-store/manifest.yaml
      config:
        bucket: local-assets

  plugins:
    media:
      source:
        path: ./plugins/media/manifest.yaml
      fileapis:
        - assets
```

For published releases, replace `source.path` with `source.ref` and `version`:

```yaml
providers:
  fileapis:
    assets:
      source:
        ref: github.com/your-org/fileapi/object-store
        version: 0.0.1
      config:
        bucket: my-assets
```

There is no `server.fileapi` selector. File API providers are reusable named
attachments, and plugins opt into them individually.
